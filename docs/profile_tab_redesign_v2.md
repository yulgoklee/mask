## 1. 왜 새로 설계하는가

### 1.1 현재 상태 진단

`profile_tab.dart` 958줄에 다음이 섞여 있음:
- 페르소나 정보 (읽기) — 결과지(/diagnosis_result)와 동일 위젯 사용 중
- 임시상태 카드 (Tier 2 — 임신·시술 등 5종)
- 오늘 컨디션 입력 영역
- 알림 설정 진입 (분산돼 있음)
- 건강 프로필 수정 진입
- 분석가 톤 잔재

문제: 매일 톤도 아니고 가끔 톤도 아니고, 진단 직후 톤·편집 톤·운영 톤이 섞임. "이 탭은 뭘 위한 곳인가?"가 불명.

> **"내가 누구인지 + 내 알림 + 내 몸 정보 진입 — 깔끔한 3블록."**

### 1.3 프로필 탭의 본질

차터 §2.1 시간축 = **앞으로**. 사용자 질문:
- "앱이 나를 어떻게 보고 있지?"
- "내 알림은 어떻게 가고 있지?"
- "뭘 바꿔야 하지?"

매일 들여다보는 곳 ✕. 가끔 신뢰 확인·변경 시 들어옴.

### 1.4 이번 사이클의 핵심 결정 (차터 §8.4)

- 임시상태 시스템 5종 (임신·영유아·꽃가루·피부시술·면역저하) **통째 제거**

## 2. 페이지 레이아웃

```
┌────────────────────────────┐
│ 프로필              ⚙️     │  타이틀(24px Bold) + 설정 진입
├────────────────────────────┤
│                            │
│ [카드 1: 페르소나]          │  결과지와 동일 위젯 4개 재활용
│                            │
│ [카드 2: 내 알림 요약]      │  신규
│                            │
│ [한 줄 링크: 내 몸 정보 →]  │  편집 진입
│                            │
└────────────────────────────┘
```

순서: 페르소나 → 알림 → 내 몸 정보 진입.
패딩·모서리·간격: 케어/리포트 탭과 동일 (DT 토큰 그대로).

## 3. 카드별 설계

### 3.1 카드 1 — 페르소나 카드 (재활용)

**구성**: 결과지(/diagnosis_result)에서 사용 중인 위젯 4개 그대로:
- ProfileStateHeader (이름 + 페르소나 뱃지)
- ThresholdCompareCard (일반 35 vs 내 기준)
- SensitivityBreakdown (5개 막대)
- SensitivityActionGuide (한 줄 메시지)

**위치**: `lib/widgets/sensitivity_widgets.dart` (이미 추출됨)

**변형**: 없음. 그대로 호출.

**이유**: 결과지에서 봤던 정보를 매번 동일하게 다시 볼 수 있어야 신뢰감 일관됨. 또 코드 중복 ✕.

### 3.2 카드 2 — 내 알림 요약 (신규)

**레이아웃**:
```
┌────────────────────────────┐
│ 🔔 내 알림                 │  카드 제목 (15px SemiBold)
│                            │
│ 매일 오전 7시 · 외출 전    │  요약 한 줄 (15px Regular)
│                            │
│ ─────────────────          │  구분선 (1px DT.border)
│                            │
│              [변경 →]      │  우측 정렬 텍스트 버튼
└────────────────────────────┘
```

**시각 스펙**:
- 카드 배경: DT.white
- 모서리: 16px
- 패딩: 20px
- 카드 제목: 15px SemiBold + 🔔 이모지
- 요약 카피: 15px Regular DT.text
- "변경 →" 버튼: 14px DT.primary 우측 정렬, 탭하면 `/notifications` push
- 구분선: 1px DT.border, 카피 ↔ 버튼 사이

**카피 룰**:
- 활성화된 알림 종류만 나열, "·"으로 구분
- 시간이 있는 항목은 "매일 오전 7시", 시간 없는 항목은 그냥 이름
- 알림 전부 꺼져있으면: `"받고 있는 알림이 없어요"`
- 예시:
  - `"매일 오전 7시 · 외출 전 · 귀가 후"`
  - `"매일 오전 7시 · 외출 전"`
  - `"외출 전"` (오전 알림 끔, 외출만)
  - `"받고 있는 알림이 없어요"` (모두 끔)

**데이터 출처**:
- NotificationSetting 모델 (`lib/data/models/notification_setting.dart`)
- profile_providers의 notificationSettingProvider 또는 동등 (단계 1 구현 시 정확한 provider 매핑)

**탭 동작**:
- 카드 어디든 탭하면 `/notifications` push (전체 카드 영역 InkWell)
- "변경 →" 버튼이 메인 affordance지만 카드 자체도 tappable

### 3.3 한 줄 링크 — 내 몸 정보 진입

**레이아웃**:
```
┌────────────────────────────┐
│ 내 몸 정보 수정         →  │  
└────────────────────────────┘
```

**시각 스펙**:
- 컨테이너: 카드 형식이지만 height 56px 고정, 1줄
- 배경: DT.white
- 모서리: 16px
- 좌측 패딩: 20px, 텍스트 15px Medium DT.text
- 우측 chevron_right 16px DT.gray
- 탭 → `/profile/edit` push (또는 `/my-body-info` — 라우터 확인)

**카피**: "내 몸 정보 수정" — 명령형 ✕, 동사형으로 "수정" 명사화하여 읽기 자연스럽게.
